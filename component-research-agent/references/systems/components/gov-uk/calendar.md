# Calendar

## Approach
GOV.UK Design System does not include a calendar grid component. As established in the Date Input documentation, GOV.UK deliberately avoids calendar pickers in favor of text-based date entry. A standalone calendar component would only be appropriate for "pick a future date" scenarios like appointment scheduling, which GOV.UK handles through department-specific implementations rather than a system-level component.

## Key Decisions
1. **No calendar component** (HIGH) — Consistent with the Date Input decision; calendar grids are avoided throughout GOV.UK's component library.
