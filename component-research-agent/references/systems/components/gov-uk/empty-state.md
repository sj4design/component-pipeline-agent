# Empty State

## Approach
GOV.UK Design System does not include a dedicated Empty State component. Government services handle empty data states through standard page content with appropriate headings and body text. The Inset Text or Panel components may be used to visually distinguish empty-state messaging, but there is no dedicated "empty state" pattern with illustration slots or call-to-action layouts.

## Key Decisions
1. **No empty-state component** (HIGH) — GOV.UK services handle "no results" and "empty data" scenarios through standard content patterns (headings + paragraphs) rather than specialized UI components. Illustrations are generally avoided in GOV.UK interfaces.
